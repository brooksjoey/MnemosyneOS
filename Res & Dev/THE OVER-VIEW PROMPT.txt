## 🎯 Rules of Engagement
1. **Modularity Doctrine**: ~37 core modules, each focused and atomic
2. **Memory Levels**: L1 (buffer) → L5 (integrator) event capture without user prompts
3. **Acceptance Blocks**: Mandatory verification steps in every module
4. **Communications Protocol**: Clear SITREP reporting after each operation
5. **Prohibited Actions**: No SSH/firewall/security modifications without explicit "ORDER:" directive

## 🧠 Memory Integration
- **L1**: Immediate buffer (current operations)
- **L2**: Short-term retention (recent events)
- **L3**: Working memory (active processing)
- **L4**: Long-term storage (consolidated knowledge)
- **L5**: Integrator (cross-module synthesis)
