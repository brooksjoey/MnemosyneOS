## 1. Operational Profile

**COMMANDER’S INTENT**  
MnemosyneOS is an AI-agnostic memory operating system. It observes, organizes, and recalls data signals while remaining modular, idempotent, and user-controlled.  

**RULES OF ENGAGEMENT (ROE)**  
- No SSH/security modifications unless explicitly ordered.  
- All modules are self-contained paste-once scripts.  
- No hidden side-effects. Explicit declaration of touched files, services, and ports.  

**MODULARITY DOCTRINE**  
- Small mission-scoped modules, ≤ ~60 lines when possible.  
- One objective per module; orchestrators only if necessary.  

**DELIVERY FORMAT**  
- Scripts self-install under `/opt/mnemo-ops/<callsign>.sh`.  
- Each module outputs: PURPOSE, INPUTS, ACTIONS, OUTPUTS, ROLLBACK, QUICKTEST.  

**QUALITY STANDARDS**  
- Robust error handling; fail fast.  
- Idempotent (no duplication on reruns).  
- Predictable, reproducible state.  

**MEMORY DOCTRINE (L1–L5)**  
- L1 buffer → L2 episodic → L3 semantic → L4 indexed → L5 integrator.  
- L5 coordinates, not bypasses.  

**COMMS (SITREP Style)**  
- Modules report SITREP on completion.  

**ACCEPTANCE CHECKS**  
- Every module outputs quick commands for verification.  

**PROHIBITED ACTIONS**  
- No monoliths.  
- No background security edits.  
- No destructive file operations unless explicitly enabled.

## 2. Addressing & Variant Menus

**Format:** `L.M[.V[.R]]`  
- **Domains (L):** B,D,R,S,C,Q,M,O,F,P,Z  
- **Runtime (R):**  
  1 = user unit  
  2 = system (current user)  
  3 = mnemo system user (preferred)  
  4 = no systemd/dev  

**Variant Menus:**  

- **B (Bootstrap)**  
  - B.1 system packages: V1=python+build, V2=+redis, V3=+postgres libs, V4=all  
  - B.2 layout/ownership: V1=/opt/mnemo, V2=~/mnemo, V3=custom  
  - B.3 venv+install: V1=git, V2=local, V3=zip, V4=wheel/sdist  
  - B.4 node toolchain: V1=npm ci, V2=yarn, V3=skip  

- **D (Database)**  
  - D.1 connectivity: V1=local, V2=remote prompt  
  - D.2 role/db: V1=local postgres, V2=prompt creds, V3=skip  
  - D.3 alembic: V1=head, V2=rev-id, V3=dry-run  
  - D.4 seed: V1=minimal, V2=examples, V3=skip  

- **R (Redis/Queues)**  
  - R.1 broker: V1=local redis://127.0.0.1, V2=prompt  
  - R.2 worker: V1=basic, V2=concurrency, V3=queues  
  - R.3 beat: V1=basic, V2=schedule path  

- **S (Services)**  
  - S.1 API: V1=8000, V2=PORT from env, V3=prompt  
  - S.2 worker svc: V1=uvicorn, V2=gunicorn, V3=custom  
  - S.3 admin/timer: V1=oneshot, V2=timer  

- **C (Collections)**  
  - C.1 declare: V1=paths, V2=tags, V3=git paths  
  - C.2 index: V1=shallow, V2=deep/entities, V3=update  
  - C.3 watch: V1=inotify, V2=scheduled  

- **Q (Query)**  
  - Q.1 keyword: V1=cURL, V2=python client, V3=vscode  
  - Q.2 hybrid: V1=important-first, V2=all, V3=prompt scope  
  - Q.3 QA: V1=concise, V2=extended  

- **M (Memory)**  
  - M.1 L1 buffer: V1=enable, V2=test, V3=tune  
  - M.2 L2 episodic: V1=enable, V2=test  
  - M.3 L3 semantic: V1=enable, V2=test

- **O (Observability)**

- **F (File Ops)**

  - F.2 dry-run organize: V1=rules file, V2=prompt

- **P (Policy)**  
  - P.1 define: V1=path, V2=tag, V3=type  
  - P.2 enforce: V1=sync, V2=continuous
